# ATM Console Simulator

Консольный симулятор банкомата на Java 11+.

## Участники

Участник 1: Асанали Мурат
Участник 2: Жанибек Мукаев
Участник 3: Ратмин Кенжеханов

## Возможности

- регистрация пользователя по номеру карты и PIN
- вход по карте и PIN с блокировкой после 3 неверных попыток
- просмотр баланса
- пополнение и снятие наличных
- перевод между пользователями системы
- история последних 10 операций
- загрузка данных при старте и сохранение в текстовый файл при выходе

## Структура проекта

- `src/atm/Main.java` - исходный код приложения
- `data/atm-data.txt` - файл хранения пользователей и транзакций

## Сборка и запуск

```bash
javac -encoding UTF-8 -d out src/atm/Main.java
java -cp out atm.Main
```

Если проект собирается в Windows-консоли, флаг `-encoding UTF-8` обязателен, потому что в исходниках используются русские строки.

## Кто какие строки кода делал

В текущей версии проекта весь код находится в файле `src/atm/Main.java`. Так как в папке проекта нет `.git`, определить авторство строк через `git blame` невозможно, поэтому ниже указано распределение по модулям и диапазонам строк текущей версии файла:

- Асанали Мурат: строки 19-279 (`Main`, `ConsoleUI`) - запуск приложения, меню, регистрация, вход, операции пользователя и сохранение данных
- Жанибек Мукаев: строки 281-431 (`AuthService`, `BankService`) - авторизация, валидация, пополнение, снятие и переводы
- Ратмин Кенжеханов: строки 433-728 (`DataStore`, `User`, `Account`, `Transaction`, `TransactionType`, исключения) - хранение данных, модели, история операций и служебные классы

## Распределение по ролям

- модели: `User`, `Account`, `Transaction`
- бизнес-логика: `AuthService`, `BankService`
- ввод/вывод и хранение: `ConsoleUI`, `DataStore`, `Main`

## Формат хранения

Данные сохраняются в обычный текстовый файл:

- `USER|...` - строка пользователя
- `TX|...` - строка транзакции

Текстовые поля кодируются в Base64, чтобы безопасно хранить русские строки и символы-разделители.
